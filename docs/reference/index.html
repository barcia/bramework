    animation

    @mixin animation($name, $duration, $function: ease-in-out, $delay: 0s, $count: infinite, $direction: alternate, $mode: fordwards, $play-state: running) { ... }

    Description

    Animation mixin with default values that you can change

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $name

    The animation name declarated in the @keyframe

    		String— none
    $duration

    The length of time that an animation should take to complete one cycle

    		Time— none
    $function

    How a CSS animation should progress over the duration of each cycle. View the docs or an example

    		Stringease-in-out
    $delay

    Specifies when the animation should start

    		Time0s
    $count

    The number of times an animation cycle should be played before stopping

    		Stringinfinite
    $direction

    Indicates whether the animation should play in reverse on alternate cycles

    		Stringalternate
    $mode

    Specifies how a CSS animation should apply styles to its target before and after it is executing

    		Stringfordwards
    $play-state

    Setermines whether an animation is running or paused

    		Stringrunning

    Example

    Usage

    .foo {
 @include animation(beat);
}

    Usage changing the duration

    .foo {
 @include animation(beat, 400ms);
}

    Keyframes declaration

    @keyframes $beat {
 from {transform: scale(1);}
 to {transform: scale(1.15);}
}
@keyframes $spin {
 0% {transform: rotate(0deg);};
 100% {transform: rotate(359deg);};
}

    Links

    • Mozilla Developer Network

    transition

    @mixin transition($property, $duration: $transition-time, $timing-function: transition-timing-function) { ... }

    Description

    Transitions

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $property

    The property who will be transitioned

    		String— none
    $duration

    The number of seconds or milliseconds a transition animation should take to complete

    		String$transition-time
    $timing-function

    How the intermediate values of the CSS properties being affected. Use ease, linear, step-end, steps(4, end), etc. View the docs or an example

    		Stringtransition-timing-function

    Example

    Simple usage

    .foo {
 @include transition(width);
}

    More than one property

    .foo {
 @include transition([width, height]);
}

    More than one property and change duration

    .foo {
 @include transition([width, height], 5s);
}

    breakpoint

    @mixin breakpoint($breakpoint) { ... }

    Description

    Breakpoint manager

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $breakpoint

    Put the size to do the breakpoint. Accepted strings [small|medium|large|extra-large]. Remember that this breakpoint mixin ir mobile first.

    		String— none

    Content

    This mixin allows extra content to be passed (through the @content directive).

    Example

    Usage

    .foo {
  width: 300px;
  @include breackpoint(medium) {
    width: 100px;
  }
}

    Output

    .foo {
  width: 300px;
}
@media (min-width:800px){
  .foo {
    width: 100px;
  }
}

    Throws

    • No value could be retrieved from #{$breakpoint}. Make sure it is defined in $breakpoints map.

    Requires

    • [variable] breakpoints
    • [variable] breakpoints

    See

    • [variable] breakpoints

    Links

    • https://www.sitepoint.com/managing-responsive-breakpoints-sass/
